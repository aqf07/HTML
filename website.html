Learn HTML

At first, of course you'll be intrigued at the idea of building your own website, and you'll definitely have a lot of questions about how to do so. The first step of creating a website is learning how to make one (obviously). Almost all websites are written in a language called HTML . HTML is not a way to format and decorate your website but a way to organize the base logic of your website. There are also countless websites out there that have tutorials on the language. The one I recommend (which, not coincidentally, is also the one I used) is W3Schools . It has an article dedicated to basically everything in HTML. Even as I'm writing this very paragraph, I'm constantly referring back W3Schools. HTML organizes an website with tags, and everything should go in a tag. The way you would write this is <tag> content </tag>. As a summary, here are the most important tags to know:

Tag Use
!DOCTYPE html Declares a file to be an html document, goes at the top of every html file
html Creates the root element of the webpage, should be the second line
body Contains all the contents of your html file, everything should go in body (or the head tag)
div Block level divider
p Represents a paragraph
h1 Creates a heading. There are 6 different headings of varying sizes
img Creates an image given a reference link
a Creates a link given an address
Of course, there are a lot, and I mean a lot, of other tags, but with these 6 tags, you can probably create a coherent website at least.

Play Around

Now, after some learning and experimentation, you probably have a rough idea of what most tags mean and how they work. However, your website probably looks something like this:

And something like that, totally sucks. Where are all the colors, the fonts, the style? Style is a tag attribute that adds styles to an element in HTML. With style, you can change colors, fonts, sizes, backgrounds, and way more. The syntax of style is

<tag style="property1: value1 property2: value2..."> content <tag>.

The style options that are most commonly used include
  • color
  • background-color
  • font-family
  • font-size
  • text-align
  • border
  • height
  • width
With these style properties, you can make your website 100 times better than the hot garbage it was before! Note that there are also other attributes like href (attaches a link to text), src (provides the source of an image), class, id, etc, but I'm getting lazy that's for you to learn on your own.

Annoying Issues

Now that you're familiar with everything, it's time to actually start building your website. First, you'd want to construct a clear structure and flow of logic. And perhaps, if you're like me, a perfectionist and borderline mentally stable, you'd realize that lining up your text boxes is the next hardest thing to waking up in the morning. If you haven't noticed already, the white bar to the left of this article is actually way thicker than the bar on the top, and at this point, I've lost all will to fix that problem --- this is something you'll just have to live with. However, there are some ways (currently) you can make this calamity a little more livable, and that is using the style options padding and border.

In short, these options allow you to change the gap between the textboxes, but from my experience, it is extremely gimmicky. There are many units you can use to determine the amount of padding and the width of your borders. The ones that I am familiar with include:

Unit Definition
vh Represents 1% of the viewing port's height
vw Represents 1% of the viewing port's width
px Represents 1 pixel
% Represents 1% of the parent element
